# 01. Plugins 개요

문서 버전: v1.0.0

## 1. 정의
Claude Code Plugin은 skills, agents, hooks, MCP servers, LSP servers 같은 확장 요소를 하나의 디렉토리 단위로 묶어 재사용하고 공유할 수 있게 해주는 배포 단위다.

## 2. 왜 사용하는가
플러그인을 사용하면 다음 문제를 해결할 수 있다.

- 여러 프로젝트에서 같은 확장 기능을 반복 복사해야 하는 문제
- 팀 단위로 skills, hooks, agents를 일관되게 배포하기 어려운 문제
- 버전 관리와 업데이트 경로가 불분명한 문제
- 마켓플레이스를 통한 배포가 필요한 문제

## 3. 플러그인이 제공하는 가치
### 3.1 재사용
하나의 플러그인을 여러 프로젝트와 사용자 범위에 적용할 수 있다.

### 3.2 배포 가능성
플러그인은 marketplace를 통해 설치·업데이트 가능한 형태로 다룰 수 있다.

### 3.3 충돌 회피
플러그인 skill은 `plugin-name:skill-name` 네임스페이스를 사용하므로 이름 충돌을 줄인다.

### 3.4 조합 가능성
하나의 플러그인 안에 skill, agent, hook, MCP, LSP를 함께 넣어 하나의 기능 묶음으로 제공할 수 있다.

## 4. 플러그인이 아닌 기능과의 관계
- CLAUDE.md: 항상 로드되는 지침
- Skills: Claude가 호출하는 지식/워크플로우
- Subagents: 전문 역할 에이전트
- Hooks: 이벤트 기반 자동화
- MCP: 외부 도구/서비스 연결
- Plugins: 위 기능들을 패키징하는 컨테이너

즉, Plugin은 자체 기능이라기보다 확장 요소를 묶어 배포하는 상위 단위다.

## 5. 대표 사용 사례
- 사내 코드 리뷰 규칙 묶음 배포
- 특정 도메인용 agent + skill + hook 조합 제공
- 팀 공통 MCP/LSP 설정 패키징
- 반복적인 운영 자동화 기능 배포
