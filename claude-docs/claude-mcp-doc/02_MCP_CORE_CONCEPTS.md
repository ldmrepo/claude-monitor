# 02. MCP 핵심 개념

## 1. MCP 서버
Claude Code가 호출하는 외부 기능 제공자다. 서버는 도구, 리소스, 프롬프트를 제공할 수 있다.

## 2. 전송 방식
- HTTP: 원격 서버 연결의 권장 방식
- SSE: 과거 방식, 현재는 deprecated
- stdio: 로컬 프로세스로 실행되는 서버

## 3. MCP가 제공하는 것
- Tools: Claude가 직접 호출하는 기능
- Resources: `@` 참조로 불러오는 외부 리소스
- Prompts: `/mcp__...` 형태의 명령으로 쓰는 프롬프트
- Channels: 외부 이벤트를 세션으로 푸시하는 확장 경로

## 4. Claude Code와의 관계
- Claude Code는 MCP 서버를 세션 시작 시 로드하거나 필요 시 탐색한다.
- 도구 설명과 스키마는 컨텍스트를 사용하므로, Tool Search가 중요한 최적화 포인트가 된다.
- MCP는 내장 도구를 대체하는 것이 아니라 외부 시스템 접속 능력을 확장한다.

## 5. 주의점
- 제3자 MCP 서버는 신뢰 가능한 공급자만 사용해야 한다.
- 프롬프트 인젝션, 과도한 출력, 과도한 권한 부여 위험이 있다.
