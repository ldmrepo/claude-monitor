# 10. 보안, 운영, 출력 제한

## 1. 보안 원칙
- 신뢰 가능한 MCP 서버만 사용
- 외부 콘텐츠를 가져오는 서버는 프롬프트 인젝션 위험 고려
- 최소 권한 원칙 적용
- 공유 설정과 개인 자격 증명 분리

## 2. 출력 제한
- 기본적으로 큰 MCP 출력은 경고가 발생할 수 있다.
- `MAX_MCP_OUTPUT_TOKENS`로 최대 허용 토큰 수를 조정할 수 있다.
- 서버 작성자는 `_meta["anthropic/maxResultSizeChars"]`로 개별 도구 임계값을 높일 수 있다.

## 3. 운영 포인트
- 대형 결과는 페이지네이션이나 요약이 유리하다.
- 과도한 출력은 세션 컨텍스트를 소모한다.
- 비자사 프록시 사용 시 Tool Search가 기본 비활성화될 수 있다.

## 4. 점검 체크리스트
- 인증 상태 정상 여부
- 서버 신뢰성
- 결과 크기
- 프록시 호환성
- 환경 변수/헤더 누락 여부
