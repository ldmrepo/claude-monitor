# 06. 작업 조정과 메시징

## 1. 공유 태스크 리스트 개념

Agent teams의 핵심은 단순 다중 세션이 아니라 **공유 태스크 리스트를 중심으로 조정되는 협업 구조**라는 점이다.

## 2. Task 상태

- pending
- in progress
- completed

## 3. 의존성 있는 태스크

어떤 태스크는 다른 태스크 완료 전까지 claim 불가 상태로 남는다.

예:
- API 설계 완료 전 구현 태스크 시작 금지
- DB schema 정리 전 migration script 작업 금지

## 4. 메시징 종류

### 4.1 특정 팀원에게 메시지
- 한 팀원에게만 전달
- follow-up, 방향 수정, 추가 요구 전달에 적합

### 4.2 broadcast
- 전체 팀원에게 동시에 전송
- 비용이 팀원 수에 비례해 증가하므로 신중히 사용

## 5. 메시징 활용 패턴

### 5.1 상호 반박
가설 기반 디버깅에서 유용하다.

### 5.2 진행 공유
한 팀원이 발견한 사실을 다른 팀원이 자신의 조사에 반영한다.

### 5.3 의존 작업 전달
선행 태스크 완료 사실을 후속 담당자가 즉시 인지한다.

## 6. 팀원 이름 전략

팀원 이름은 명확하고 역할 중심으로 짓는 것이 좋다.

예:
- ux-reviewer
- architecture-lead
- devil-advocate
- backend-owner
- test-owner

이렇게 해야 이후 자연어 지시에서 특정 팀원을 안정적으로 지목할 수 있다.

## 7. 운영 팁

- broad coordination은 리드가 담당하게 한다.
- narrow clarification은 팀원 직접 메시징을 활용한다.
- broadcast는 정책 공지, 전원 중단, 전원 기준 변경 같은 경우에 한정한다.
