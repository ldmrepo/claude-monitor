# 02. Agent Teams vs Subagents

## 1. 공통점

- 둘 다 병렬 작업을 지원한다.
- 둘 다 별도 컨텍스트를 활용한다.
- 둘 다 메인 세션의 부담을 줄일 수 있다.

## 2. 가장 큰 차이

Subagent는 **메인 에이전트의 보조 실행 단위**이고, Agent team은 **여러 독립 Claude Code 세션의 협업 구조**이다.

## 3. 비교표

| 항목 | Subagent | Agent team |
|---|---|---|
| 컨텍스트 | 독립 컨텍스트, 결과만 호출자에게 반환 | 각 팀원이 완전히 독립된 세션 |
| 통신 | 메인 에이전트에게만 보고 | 팀원 간 직접 메시징 가능 |
| 조정 방식 | 메인 에이전트가 전담 | 공유 태스크 리스트 기반 자율 조정 |
| 적합한 작업 | 결과만 중요하고 빠른 병렬 보조 작업 | 토론, 상호검증, 협업이 필요한 복합 작업 |
| 토큰 비용 | 상대적으로 낮음 | 상대적으로 높음 |

## 4. Subagent가 더 적합한 경우

- 코드 검색 후 요약만 필요할 때
- 파일 다수를 읽되 결과만 받으면 될 때
- 메인 세션이 전체 흐름을 계속 쥐고 있어야 할 때

## 5. Agent team이 더 적합한 경우

- 팀원끼리 결과를 공유해야 할 때
- 각자 다른 가설을 시험해야 할 때
- 작업을 여러 사람처럼 분업해야 할 때
- 리드와 팀원 간, 팀원 상호 간 조정이 중요할 때

## 6. 선택 기준

### 6.1 질문 하나로 구분
**작업자들끼리 서로 대화해야 하는가?**

- 아니오 → Subagent 우선
- 예 → Agent team 우선

### 6.2 비용 관점
Agent team은 각 팀원이 별도 Claude 인스턴스로 동작하므로 토큰 비용이 크게 증가한다. 따라서 단순 병렬화보다 **협업적 병렬화**가 진짜 필요한 경우에만 사용하는 것이 좋다.
