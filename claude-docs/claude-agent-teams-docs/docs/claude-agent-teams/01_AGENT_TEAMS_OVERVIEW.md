# 01. Agent Teams 개요

## 1. 정의

Agent teams는 여러 Claude Code 세션을 하나의 협업 팀으로 묶어 병렬 작업을 수행하게 하는 실험적 기능이다. 한 세션은 **team lead**로 동작하며 팀 생성, 작업 분해, 작업 할당, 결과 종합을 담당한다. 나머지 세션은 **teammate**로 동작하며 각자 독립적인 컨텍스트 윈도우에서 작업한다.

## 2. 왜 필요한가

단일 세션은 순차적으로 사고하고 실행하는 데 강하지만, 다음과 같은 경우에는 병렬 탐색이 더 유리하다.

- 서로 다른 관점의 동시 조사
- 독립 모듈 병렬 개발
- 경쟁 가설 기반 디버깅
- 프론트엔드/백엔드/테스트 등 계층 분리 작업

## 3. 핵심 특징

### 3.1 팀 리드 중심 조정
- 팀 리드가 팀 전체를 관리한다.
- 팀원 생성, 작업 관리, 결과 수집, 최종 보고를 담당한다.

### 3.2 팀원 간 직접 통신
- 팀원은 리드에게만 보고하는 구조가 아니라 서로 메시지를 주고받을 수 있다.
- 따라서 독립 조사 결과를 상호 검증하거나 반박할 수 있다.

### 3.3 공유 작업 목록
- 작업(Task)은 공유 리스트에 저장된다.
- 팀원은 작업을 직접 claim 하거나 리드에게 할당받을 수 있다.
- 의존성이 있는 작업은 선행 작업 완료 후에만 진행된다.

### 3.4 독립된 컨텍스트
- 각 팀원은 별도 컨텍스트 윈도우를 가진다.
- 리드의 대화 이력은 자동 승계되지 않는다.
- 대신 프로젝트 수준 컨텍스트(CLAUDE.md, MCP, Skills)는 일반 세션처럼 로드된다.

## 4. 일반적인 처리 흐름

1. 사용자가 팀 생성을 요청한다.
2. 리드가 팀 구조를 설계한다.
3. 팀원이 생성된다.
4. 리드가 작업을 쪼개어 태스크 리스트에 등록한다.
5. 팀원이 병렬로 작업한다.
6. 팀원 간 메시지 교환 및 결과 검증이 일어난다.
7. 리드가 결과를 수집·종합한다.
8. 팀을 종료하고 정리한다.

## 5. 적합한 작업 유형

- 대규모 조사/리뷰
- 신기능 설계 및 분업 구현
- 다가설 디버깅
- 다계층 변경 작업

## 6. 부적합한 작업 유형

- 강하게 순차적인 작업
- 동일 파일 동시 수정이 많은 작업
- 조정보다 실행 자체가 더 단순한 작업
- 토큰 비용을 최소화해야 하는 작업
