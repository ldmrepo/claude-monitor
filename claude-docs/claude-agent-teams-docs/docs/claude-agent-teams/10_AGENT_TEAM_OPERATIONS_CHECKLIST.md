# 10. 운영 체크리스트

## 1. 도입 전 체크

- [ ] Claude Code 버전이 요구사항 이상인가
- [ ] `CLAUDE_CODE_EXPERIMENTAL_AGENT_TEAMS=1` 설정 완료
- [ ] 표시 모드(in-process / split panes) 결정
- [ ] tmux 또는 iTerm2 환경 준비 여부 확인
- [ ] 권한 정책 사전 점검

## 2. 팀 생성 체크

- [ ] 팀 목표가 명확한가
- [ ] 역할 간 중복이 적은가
- [ ] 팀 규모가 3~5명 수준에서 시작되는가
- [ ] task decomposition이 가능한 문제인가

## 3. 실행 중 체크

- [ ] task list가 정상적으로 생성되었는가
- [ ] 팀원들이 서로 다른 작업을 맡고 있는가
- [ ] 동일 파일 충돌 위험이 없는가
- [ ] 리드가 직접 구현보다 조정에 집중하는가
- [ ] 필요 시 팀원 간 메시징이 활용되는가

## 4. 종료 전 체크

- [ ] 모든 중요 태스크 상태 확인
- [ ] stuck / blocked task 여부 확인
- [ ] 필요한 결과가 종합되었는가
- [ ] 활성 팀원을 먼저 shutdown 했는가
- [ ] 마지막으로 cleanup을 리드가 실행했는가

## 5. 추천 운영 원칙

1. 작은 팀으로 시작한다.
2. 조사/리뷰부터 적용한다.
3. 파일 소유권을 분리한다.
4. 리드는 조정자 역할을 유지한다.
5. cleanup을 규율화한다.
