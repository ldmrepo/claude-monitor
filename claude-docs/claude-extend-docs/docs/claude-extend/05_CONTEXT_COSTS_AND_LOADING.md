# 컨텍스트 비용과 로딩 방식
문서 ID: CC-EXT-CONTEXT
버전: v1.0.0

## 1. 기본 원칙
확장 기능은 기능성뿐 아니라 컨텍스트 비용을 발생시킨다.
과도한 항상-로딩은 비용과 노이즈를 증가시킨다.

## 2. 기능별 로딩 시점
### CLAUDE.md
- 세션 시작 시 전체 로드
- 비용: 높음 / 항상 반영

### Skills
- 기본: 설명만 시작 시 로드, 전체는 사용 시 로드
- `disable-model-invocation: true`면 수동 호출 전까지 0 비용

### MCP
- 세션 시작 시 도구 이름 로드
- 전체 스키마는 필요 시 지연 로드

### Subagents
- 스폰 시 독립 컨텍스트 생성
- 메인 세션 비용과 격리

### Hooks
- 트리거될 때 외부 실행
- 기본적으로 컨텍스트 비용 0
- 단, `additionalContext` 또는 메시지 반환 시 비용 발생

## 3. 비용 절감 전략
- CLAUDE.md는 짧게 유지
- 세부 규칙은 Rules로 분리
- 대형 참조 문서는 Skills로 이동
- 대량 조사 작업은 Subagent 사용
- side effect는 Hooks로 처리
- 안 쓰는 MCP 서버는 제거
