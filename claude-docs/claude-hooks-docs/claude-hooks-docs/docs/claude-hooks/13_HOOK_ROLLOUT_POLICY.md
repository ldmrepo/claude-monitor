# Hook 적용 정책
문서 ID: CC-HOOKS-ROLLOUT
버전: v1.1.0

## 1. 목적
Hook을 개인 실험 단계에서 프로젝트 공통 정책 단계로 안정적으로 승격하기 위한 운영 정책을 정의한다.

## 2. 적용 레벨
### 2.1 개인 레벨
- 위치: `~/.claude/settings.json`
- 대상: 개인 생산성, 알림, 로컬 단축 자동화

### 2.2 프로젝트 레벨
- 위치: `.claude/settings.json`
- 대상: 공통 품질 게이트, 위험 명령 차단, 민감 파일 보호

### 2.3 로컬 프로젝트 레벨
- 위치: `.claude/settings.local.json`
- 대상: 개인 실험, 개발 장비 전용 경로, 임시 정책

### 2.4 관리 정책 레벨
- 대상: 조직 공통 보안/감사 정책
- 비고: 사용자/프로젝트 설정으로 비활성화할 수 없는 정책만 배치

## 3. 승격 기준
개인 훅을 프로젝트 훅으로 올리려면 아래를 만족해야 한다.
- 반복 사용 빈도가 높다
- 팀 전체에 동일 가치가 있다
- 차단 기준이 명확하다
- 오탐 비용이 낮다
- 디버깅이 가능하다

## 4. 권장 배포 순서
1. 개인 훅으로 실험
2. `.claude/settings.local.json`에서 안정화
3. 프로젝트 훅으로 공유
4. 필요 시 plugin으로 패키징
5. 최종적으로 managed policy 고려

## 5. 금지 사항
- 미검증 command hook을 곧바로 프로젝트 공통 정책으로 배포하지 않는다
- 조직 강제 정책을 CLAUDE.md만으로 대체하지 않는다
- 차단 불가능한 이벤트에 enforcement 로직을 기대하지 않는다
